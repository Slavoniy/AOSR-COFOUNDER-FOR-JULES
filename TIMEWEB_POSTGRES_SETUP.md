# Как создать базу данных PostgreSQL в Timeweb и подключить её к проекту

Чтобы ваше приложение (личный кабинет) сохраняло пользователей и проекты надежно, мы переводим его на базу данных PostgreSQL, которая будет работать в Timeweb.

Следуйте этой пошаговой инструкции, чтобы создать базу данных и получить данные для подключения.

## Шаг 1: Авторизация в панели управления Timeweb
1. Зайдите в свой личный кабинет на сайте [Timeweb Cloud](https://timeweb.cloud/) (или обычный виртуальный хостинг, если вы используете его, но интерфейс Cloud удобнее для баз данных).
2. Введите свой логин и пароль.

## Шаг 2: Создание Базы Данных PostgreSQL
1. В левом боковом меню найдите раздел **"Базы данных"** (Databases).
2. Нажмите кнопку **"Создать базу данных"** (или "Добавить").
3. Выберите тип СУБД: **PostgreSQL**.
4. Выберите версию PostgreSQL (рекомендуется **14** или **15**).
5. Задайте параметры:
   - **Имя базы данных** (например, `stroydoc_db`)
   - **Имя пользователя** (например, `stroydoc_user`)
   - **Пароль**: сгенерируйте надежный пароль и **ОБЯЗАТЕЛЬНО сохраните его** себе в блокнот.
6. Выберите подходящий тариф (для старта подойдет минимальный).
7. Нажмите **"Создать"** и дождитесь, пока база данных будет инициализирована (статус изменится на "Работает" или "Активна").

## Шаг 3: Получение данных для подключения (DATABASE_URL)
Когда база данных создана, вам нужно собрать строку подключения, которая выглядит так:
`postgresql://ПОЛЬЗОВАТЕЛЬ:ПАРОЛЬ@ХОСТ:ПОРТ/ИМЯ_БАЗЫ`

Где взять эти данные в панели Timeweb:
- **ПОЛЬЗОВАТЕЛЬ** — это имя пользователя, которое вы создали (например, `stroydoc_user`).
- **ПАРОЛЬ** — пароль, который вы сохранили.
- **ХОСТ** — IP-адрес или доменное имя вашего сервера баз данных (отображается в настройках БД в Timeweb, например, `db12345.timeweb.cloud` или просто IP).
- **ПОРТ** — стандартный порт для PostgreSQL — `5432` (если в панели не указан другой).
- **ИМЯ_БАЗЫ** — имя базы данных (например, `stroydoc_db`).

**Пример готовой строки подключения:**
`postgresql://stroydoc_user:MySecretPass123!@12.34.56.78:5432/stroydoc_db`

*(Если база данных находится на том же сервере, что и ваш сайт, то ХОСТ может быть `localhost` или `127.0.0.1`)*.

## Шаг 4: Добавление DATABASE_URL в проект
1. Откройте корневую папку вашего проекта на сервере (или у себя на компьютере).
2. Найдите файл `.env` (если его нет, скопируйте файл `.env.example` и назовите его `.env`).
3. Добавьте в файл `.env` новую строку:

```env
DATABASE_URL="ваша_строка_подключения_здесь"
```

*Пример:*
```env
JWT_SECRET="ваш_надежный_секретный_ключ"
DATABASE_URL="postgresql://stroydoc_user:MySecretPass123!@12.34.56.78:5432/stroydoc_db"
```

## Шаг 5: Перезапуск сервера
После добавления `DATABASE_URL` вам нужно перезапустить ваш Node.js сервер:
```bash
npm run dev
# или как вы запускаете его на сервере (pm2 restart stroydoc, systemctl restart... и т.д.)
```

При запуске приложение автоматически подключится к вашей базе данных PostgreSQL и создаст в ней нужные таблицы (`users` и `projects`).

Теперь регистрация и вход будут работать надежно, а проекты будут сохраняться навсегда!
